# tinyriscv 处理器指令集文档

## 概述

tinyriscv 是一个单核32位小型RISC-V处理器，支持 **RV32IM** 指令集架构，同时支持部分 **CSR** 指令。采用三级流水线（取指→译码→执行）设计，所有ALU运算（除乘法除法外）均在同一周期内完成。乘法指令在ex模块中以组合逻辑实现，除法指令由独立的 `div` 模块以试商法实现（至少需要33个时钟周期）。

---

## 一、运算类指令 (RV32I 基础整数运算)

### 1.1 I型立即数运算指令

**opcode = `0010011` (7'b0010011)**

指令格式：`opcode[6:0] | rd[11:7] | funct3[14:12] | rs1[19:15] | imm[31:20]`

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **ADDI** | `000` | rd ← rs1 + imm (符号扩展)。将寄存器rs1的值与12位立即数符号扩展后相加，结果写入rd。 |
| **SLTI** | `010` | rd ← (signed(rs1) < signed(imm)) ? 1 : 0。有符号比较，若rs1小于立即数则rd置1，否则置0。 |
| **SLTIU** | `011` | rd ← (rs1 < imm(无符号扩展)) ? 1 : 0。无符号比较，若rs1小于立即数则rd置1，否则置0。 |
| **XORI** | `100` | rd ← rs1 ^ imm。按位异或。 |
| **ORI** | `110` | rd ← rs1 \| imm。按位或。 |
| **ANDI** | `111` | rd ← rs1 & imm。按位与。 |
| **SLLI** | `001` | rd ← rs1 << shamt (shamt = inst[24:20])。逻辑左移，移位量由立即数低5位指定。 |
| **SRI** | `101` | `inst[30]=0` 时：**SRLI** — rd ← rs1 >> shamt (逻辑右移)； `inst[30]=1` 时：**SRAI** — rd ← rs1 >>> shamt (算术右移，高位由符号位填充)。 |

### 1.2 R型寄存器-寄存器运算指令

**opcode = `0110011` (7'b0110011), funct7 = `0000000` 或 `0100000`**

指令格式：`opcode[6:0] | rd[11:7] | funct3[14:12] | rs1[19:15] | rs2[24:20] | funct7[31:25]`

| 指令 | funct3 | funct7 | 行为描述 |
|------|--------|--------|----------|
| **ADD**  | `000` | `0000000` | rd ← rs1 + rs2。加法。 |
| **SUB**  | `000` | `0100000` | rd ← rs1 - rs2。减法（由 inst[30]=1 区分）。 |
| **SLL**  | `001` | `0000000` | rd ← rs1 << rs2[4:0]。逻辑左移。 |
| **SLT**  | `010` | `0000000` | rd ← (signed(rs1) < signed(rs2)) ? 1 : 0。有符号比较置位。 |
| **SLTU** | `011` | `0000000` | rd ← (rs1 < rs2) ? 1 : 0。无符号比较置位。 |
| **XOR**  | `100` | `0000000` | rd ← rs1 ^ rs2。按位异或。 |
| **SRL**  | `101` | `0000000` | rd ← rs1 >> rs2[4:0]。逻辑右移（由 inst[30]=0 区分）。 |
| **SRA**  | `101` | `0100000` | rd ← rs1 >>> rs2[4:0]。算术右移（由 inst[30]=1 区分）。 |
| **OR**   | `110` | `0000000` | rd ← rs1 \| rs2。按位或。 |
| **AND**  | `111` | `0000000` | rd ← rs1 & rs2。按位与。 |

## 二、乘法与除法指令 (RV32M)

**opcode = `0110011` (7'b0110011), funct7 = `0000001`**

指令格式：与R型指令相同

乘法在ex模块中通过组合逻辑 `mul_op1 * mul_op2` 产生64位中间结果。除法在div模块中以试商法迭代完成，需要多个时钟周期。

### 2.1 乘法指令

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **MUL** | `000` | rd ← (rs1 × rs2)[31:0]。有符号乘法，取低32位结果。 |
| **MULH** | `001` | rd ← (signed(rs1) × signed(rs2))[63:32]。有符号×有符号，取高32位结果。注意：操作数为负时先取绝对值相乘，结果根据符号情况取反。 |
| **MULHSU** | `010` | rd ← (signed(rs1) × unsigned(rs2))[63:32]。有符号×无符号，取高32位。rs1为负时取绝对值参与乘法，结果取反。 |
| **MULHU** | `011` | rd ← (unsigned(rs1) × unsigned(rs2))[63:32]。无符号×无符号，取高32位。 |

### 2.2 除法与取余指令

除法使用试商法实现，运算期间流水线暂停（hold），需要至少33个时钟周期。跳转到下一条指令时流水线才继续。

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **DIV** | `100` | rd ← signed(rs1) ÷ signed(rs2)。有符号除法（商向0舍入）。除法器输出64位结果（高32位余数，低32位商），rd取低32位（商）。 |
| **DIVU** | `101` | rd ← rs1 ÷ rs2。无符号除法。 |
| **REM** | `110` | rd ← signed(rs1) mod signed(rs2)。有符号取余。rd取高32位（余数）。 |
| **REMU** | `111` | rd ← rs1 mod rs2。无符号取余。 |

## 三、加载指令 (Load)

**opcode = `0000011` (7'b0000011)**

指令格式：`opcode[6:0] | rd[11:7] | funct3[14:12] | rs1[19:15] | imm[31:20]`

计算地址 = rs1 + sign_extend(imm)，发出读内存请求，从内存返回数据中截取对应字节/半字并做符号/零扩展。

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **LB** | `000` | rd ← sign_extend(Mem[rs1+imm][7:0])。加载字节并符号扩展到32位。 |
| **LH** | `001` | rd ← sign_extend(Mem[rs1+imm][15:0])。加载半字并符号扩展。 |
| **LW** | `010` | rd ← Mem[rs1+imm][31:0]。加载32位字。 |
| **LBU** | `100` | rd ← zero_extend(Mem[rs1+imm][7:0])。加载字节并零扩展到32位。 |
| **LHU** | `101` | rd ← zero_extend(Mem[rs1+imm][15:0])。加载半字并零扩展。 |

### 非对齐访问处理

所有加载指令均在ex模块中通过 `mem_raddr_index`（地址低2位）从32位内存读取数据中选取正确的字节/半字：
- **LB/LBU**：4路选择（按字节偏移0/1/2/3）
- **LH/LHU**：2路选择（按半字偏移0或2）
- **LW**：直接返回32位数据

## 四、存储指令 (Store)

**opcode = `0100011` (7'b0100011)**

指令格式：`opcode[6:0] | imm[4:0]=inst[11:7] | funct3[14:12] | rs1[19:15] | rs2[24:20] | imm[11:5]=inst[31:25]`

计算地址 = rs1 + sign_extend({inst[31:25], inst[11:7]})。采用读-修改-写方式处理非对齐：先读出目标地址的32位字，修改对应字节/半字后再写回。

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **SB** | `000` | Mem[rs1+imm][7:0] ← rs2[7:0]。存储字节。先读出目标字，用rs2低8位替换对应字节后写回。 |
| **SH** | `001` | Mem[rs1+imm][15:0] ← rs2[15:0]。存储半字。先读出目标字，用rs2低16位替换对应半字后写回。 |
| **SW** | `010` | Mem[rs1+imm][31:0] ← rs2。存储32位字，直接写入。 |

## 五、分支指令 (Branch)

**opcode = `1100011` (7'b1100011)**

指令格式：`opcode[6:0] | imm[11]=inst[7] | imm[4:1]=inst[11:8] | funct3[14:12] | rs1[19:15] | rs2[24:20] | imm[10:5]=inst[30:25] | imm[12]=inst[31]`

分支目标地址 = PC + sign_extend({inst[31], inst[7], inst[30:25], inst[11:8], 1'b0})。（注意：该处理器的立即数拼接顺序与标准RISC-V略有不同，需要参考inst_i的具体位域）

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **BEQ** | `000` | 若 rs1 == rs2 则跳转到目标地址。比较相等转移。 |
| **BNE** | `001` | 若 rs1 != rs2 则跳转到目标地址。比较不等转移。 |
| **BLT** | `100` | 若 signed(rs1) < signed(rs2) 则跳转。有符号小于转移。 |
| **BGE** | `101` | 若 signed(rs1) >= signed(rs2) 则跳转。有符号大于等于转移。 |
| **BLTU** | `110` | 若 rs1 < rs2（无符号）则跳转。无符号小于转移。 |
| **BGEU** | `111` | 若 rs1 >= rs2（无符号）则跳转。无符号大于等于转移。 |

---

## 六、跳转指令 (Jump)

| 指令 | opcode | 行为描述 |
|------|--------|----------|
| **JAL** | `1101111` | rd ← PC+4；跳转到 PC + sign_extend({inst[31], inst[19:12], inst[20], inst[30:21], 1'b0})。保存返回地址到rd，无条件跳转。 |
| **JALR** | `1100111` | rd ← PC+4；跳转到 (rs1 + sign_extend(inst[31:20])) & ~1。寄存器间接跳转，保存返回地址。 |

---

## 七、立即数操作指令

| 指令 | opcode | 行为描述 |
|------|--------|----------|
| **LUI** | `0110111` | rd ← {inst[31:12], 12'b0}。将20位立即数加载到rd的高20位，低12位清零。 |
| **AUIPC** | `0010111` | rd ← PC + {inst[31:12], 12'b0}。将20位立即数左移12位后加上当前PC值，结果写入rd。 |

## 八、CSR指令 (控制和状态寄存器)

**opcode = `1110011` (7'b1110011)**

指令格式：`opcode[6:0] | rd[11:7] | funct3[14:12] | rs1/uimm[19:15] | csr_addr[31:20]`

该处理器支持的CSR寄存器地址：

| CSR名称 | 地址 | 描述 |
|---------|------|------|
| **mstatus** | `0x300` | 机器状态寄存器（bit[3]控制全局中断使能） |
| **mie** | `0x304` | 机器中断使能寄存器 |
| **mtvec** | `0x305` | 机器陷阱向量基址寄存器 |
| **mepc** | `0x341` | 机器异常程序计数器 |
| **mcause** | `0x342` | 机器异常原因寄存器 |
| **mscratch** | `0x340` | 机器暂存寄存器 |
| **cycle** | `0xc00` | 周期计数器低32位 |
| **cycleh** | `0xc80` | 周期计数器高32位 |

### 8.1 寄存器操作CSR指令

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **CSRRW** | `001` | rd ← CSR[csr_addr]；CSR[csr_addr] ← rs1。原子读后写。 |
| **CSRRS** | `010` | rd ← CSR[csr_addr]；CSR[csr_addr] ← CSR[csr_addr] \| rs1。读后置位。 |
| **CSRRC** | `011` | rd ← CSR[csr_addr]；CSR[csr_addr] ← CSR[csr_addr] & (~rs1)。读后清零。 |

### 8.2 立即数操作CSR指令

| 指令 | funct3 | 行为描述 |
|------|--------|----------|
| **CSRRWI** | `101` | rd ← CSR[csr_addr]；CSR[csr_addr] ← zero_extend(uimm)。读后写立即数（uimm = inst[19:15]，零扩展到32位）。 |
| **CSRRSI** | `110` | rd ← CSR[csr_addr]；CSR[csr_addr] ← CSR[csr_addr] \| zero_extend(uimm)。读后置位立即数。 |
| **CSRRCI** | `111` | rd ← CSR[csr_addr]；CSR[csr_addr] ← CSR[csr_addr] & (~zero_extend(uimm))。读后清零立即数。 |

## 九、其他指令

| 指令 | opcode | 行为描述 |
|------|--------|----------|
| **NOP** | `0000001` | 空操作。编码为 `0x00000001`（自定义编码，非标准RISC-V的NOP）。不执行任何操作，流水线正常前进。 |
| **FENCE** | `0001111` | 内存屏障指令。当前实现中视为NOP + 跳转到下一条指令（PC+4），即仅保证指令顺序，不执行实际的缓存/内存同步操作。 |

## 关键实现说明

1. **流水线结构**：三级流水线——取值(IF)→译码(ID)→执行(EX)。RISC-V Internal Bus (RIB) 总线连接各外设。

2. **ALU运算**：ADD/SUB通过 `op1_add_op2_res = op1_i + op2_i` 和 `reg_wdata = op1_i - op2_i` 实现加减；比较指令使用 `$signed()` 系统函数做有符号比较；移位指令使用Verilog的 `>>`、`<<` 运算符。

3. **乘法实现**：通过64位临时变量 `mul_temp = mul_op1 * mul_op2` 实现，MUL取低32位，MULH/MULHU/MULHSU取高32位。负数处理采用先取绝对值相乘再根据符号条件取反的策略。

4. **除法实现**：独立 `div.v` 模块，采用试商法迭代实现，每次运算至少需要33个时钟周期，期间通过 `hold_flag` 暂停流水线。

5. **CSR寄存器**：在 `csr_reg.v` 模块中实现，支持8个CSR寄存器，读写操作通过ex模块的CSR读写信号完成。全局中断使能由mstatus[3]控制。

6. **中断支持**：支持timer0中断（中断号 `0x01`，跳转地址 `0x04`），通过 `int_assert_i` 信号触发，中断响应时暂停当前指令的寄存器写入和内存访问。

7. **非对齐内存访问**：加载指令通过地址低2位从32位数据中选取对应字节/半字；存储指令采用读-修改-写方式处理SB/SH的非对齐。地址对齐由硬件自动处理。
